# store_rejection_flow

## 標準フロー
1. 指摘内容を `文言 / 機能 / 素材 / 手続き` に分類する
2. 再現条件と影響範囲を確認し、修正方針を決める
3. 修正を PR 化し、テストと再確認を行う
4. 再提出ログを更新し、同じ指摘の再発防止策を追記する

## SLA 目安
- 文言 / 素材差し替え: `24時間以内` に再提出判断
- 軽微な機能修正: `72時間以内` に再提出判断
- 広範囲の機能修正: 別途公開可否を再判定
