# 5-1. 商品設計

- Source: `TASKS.md`

## T01 [P0] カスタム版の提供内容を固定する

- タスク内容:
  - カスタム版の提供内容を固定する
- 目的:
  - 申込前に納品範囲を明確化するため。
- 草案:
  - 提供内容: FIT分析 / ヒアリング反映 / 設定値一覧 / 根拠説明 / カスタムコード。
  - 納品形式: PDFまたはMarkdownレポート + 設定値一覧 + コード文字列。
  - 提供外: 医療判断、レース当日のリアルタイムコーチング。
  - 注意点: 「何をもらえるか」を先に示す構成にする。
- 結果:
  - `カスタム版の提供内容は「FIT分析 / ヒアリング反映 / 設定値一覧 / 根拠説明 / カスタムコード」に固定し、納品形式は「レポート + 設定値一覧 + コード文字列」で確定する。`

## T02 [P0] カスタム版の料金を決める

- タスク内容:
  - カスタム版の料金を決める
- 目的:
  - 申込導線で迷わず判断できる料金表示にするため。
- 草案:
  - 採用料金: `標準プラン 4,980円（税込）`。
  - 料金に含む: 初回分析、設定提案、カスタムコード、14日以内1回再調整。
  - 追加料金: 2回目以降は初回納品後60日以内・1回1,500円で受付。
  - 注意点: 税込価格と追加料金条件を同じ場所に記載する。
- 結果:
  - `料金は「標準プラン4,980円（税込）」で確定し、追加料金は「2回目以降は初回納品後60日以内・1回1,500円」で運用する。`

## T03 [P0] 修正対応ルールを決める

- タスク内容:
  - 修正対応ルールを決める
- 目的:
  - 納品後の対応範囲を明確にして運用負荷を管理するため。
- 草案:
  - 無料再調整: 納品後14日以内、1回まで。
  - 有料再調整: 2回目以降は初回納品後60日以内・1回1,500円で受付。
  - 対象外: 大幅な前提変更（別レース距離、体調要因の大幅変化）は追加料金。
  - 注意点: 無料範囲を数値で定義し、判断のブレを防ぐ。
- 結果:
  - `修正対応ルールは「無料再調整: 納品後14日以内1回まで」「有料再調整: 2回目以降は初回納品後60日以内・1回1,500円」「大幅な前提変更は追加料金」で確定する。`

## T04 [P1] 納品までの目標日数を決める

- タスク内容:
  - 納品までの目標日数を決める
- 目的:
  - 申込時に現実的な期待納期を提示するため。
- 草案:
  - 目標納期: 必要資料受領から `3営業日以内` に初回納品。
  - 繁忙時上限: `5営業日` を超える場合は受付時点で告知。
  - 再調整納期: 依頼受領から `2営業日以内`。
  - 注意点: 納期はカレンダー日ではなく営業日で明記する。
- 結果:
  - `納品目標日数は「初回納品: 必要資料受領から3営業日以内」「繁忙時上限: 5営業日（超過見込み時は受付時告知）」「再調整: 依頼受領から2営業日以内」で確定する。`

## 追加改善（マーケ視点）

- T01: 提供内容を `何がいつ届くか` の時系列で提示し、申込前不安を下げる。
- T02: 料金は `比較軸`（通常版との差分価値）を同じ表に置いて納得感を高める。
- T03: 修正対応ルールに `想定質問例` を付け、境界説明コストを削減する。
- T04: 納期は `最短/標準/繁忙` の3段階表示にし、期待値管理を強化する。
