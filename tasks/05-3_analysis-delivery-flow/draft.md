## T02 [P0] 指標から7項目へ変換するルール表を作る

- タスク内容:
  - 指標から7項目へ変換するルール表を作る
- 目的:
  - 定量指標を利用者向けの行動提案に変換するため。
- 草案:
  - 7項目: ペース安定性 / 心拍余裕 / 補給タイミング / 脱水兆候 / 後半維持力 / 失速リスク / 推奨戦略。

  - 判定ランク: `良好 / 注意 / 要改善` の3段階。
  - 注意点: 主観コメントはルール判定の後に補足として書く。

## T03 [P0] 根拠説明テンプレートを作る

- タスク内容:
  - 根拠説明テンプレートを作る
- 目的:
  - 提案内容の納得感を上げ、再現行動につなげるため。
- 草案:
  - テンプレ構成: `観測事実` → `解釈` → `設定提案` → `期待効果`。
  - 記載例: `30km以降のペース低下6.8%を確認。補給間隔を35→30分に短縮提案。`
  - 禁止事項: 根拠数値なしの断定、医療判断表現。
  - 注意点: 事実と提案を1対1で紐づける。

## T04 [P0] カスタムコード生成仕様を固定する

- タスク内容:
  - カスタムコード生成仕様を固定する
- 目的:

- 草案:
  - コード仕様: 設定キーを順序固定で連結し、チェックサムを末尾に付与。

  - 検証: 生成後にサンプル端末設定へ入力し、読込成功を確認。
  - 注意点: 仕様変更時はバージョンを上げ、旧コード互換性を明示する。

## T06 [P1] 再調整フローを作る

- タスク内容:
  - 再調整フローを作る
- 目的:
  - 再調整依頼を短時間で処理するため。
- 草案:
  - 受付条件: 納品後14日以内、1回まで無料。
  - 必要情報: 変更理由 / 新しい目標 / 再計測FIT（任意）。
  - 対応手順: 受付確認 → 再評価 → 差分設定値納品（2営業日以内）。
  - 注意点: 初回納品との差分を明示して再調整内容を説明する。
