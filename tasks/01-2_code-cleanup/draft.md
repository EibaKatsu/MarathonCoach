## T03 [P1] バージョン番号の運用ルールを決める

### 成果物草案
- タスク内容:
  - バージョン番号の運用ルールを決める
- 目的:
  - 判断を先送りせず、実装と公開準備の手戻りを減らすため。
- 草案:
  - 候補を2〜3案に絞り、採用案を1つ選ぶ。
  - 採用理由を「ユーザー価値・運用コスト・リスク」で1行ずつ書く。
  - 検討材料: 例 `0.9.0-beta` → `1.0.0`
