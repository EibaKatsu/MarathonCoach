# 3-2. パッケージ作成と申請

- Source: `TASKS.md`

## T01 [P0] manifest の内容を最終確認する

### 成果物草案
- タスク内容:
  - manifest の内容を最終確認する
- 目的:
  - 仕様漏れや不具合を公開前に防ぐため。
- 草案:
  - 確認観点を正常系/異常系/境界値の3つに分ける。
  - 結果を OK/NG と原因メモ付きで記録する。

## T03 [P0] ストア申請フォームへ登録する

## T05 [P1] 審査差し戻し時の修正フローを決める

### 成果物草案
- タスク内容:
  - 審査差し戻し時の修正フローを決める
- 目的:
  - 判断を先送りせず、実装と公開準備の手戻りを減らすため。
- 草案:
  - 候補を2〜3案に絞り、採用案を1つ選ぶ。
  - 採用理由を「ユーザー価値・運用コスト・リスク」で1行ずつ書く。
