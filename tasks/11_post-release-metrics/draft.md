# 11. 公開後に追う指標

- Source: `TASKS.md`

## T01 [P1] 公開後に確認するKPIを決める

- タスク内容:
  - 公開後に確認するKPIを決める
- 目的:
  - 公開後の改善優先度をデータで判断するため。
- 草案:
  - 主要KPI: ダウンロード数、レビュー件数、初回設定離脱率、カスタム版申込率、再調整率。
  - 目標値(初月): ダウンロード100件、レビュー10件、申込率3%以上。
  - 確認頻度: 週次で短報、月次で改善計画を更新。
  - 注意点: KPIは「取れる数値」だけでなく「意思決定に使う数値」に絞る。

## T02 [P1] ユーザーフィードバック記録場所を決める

- タスク内容:
  - ユーザーフィードバック記録場所を決める
- 目的:
  - フィードバック散逸を防ぎ、改善履歴を追跡可能にするため。
- 草案:
  - 記録先: `tasks/feedback/feedback_log.csv`（または同等の単一管理表）。
  - 必須列: 受信日 / 媒体 / 機種 / 内容要約 / 重要度 / 対応状況 / 反映版。
  - 運用ルール: 受信当日中に一次分類し、週次で優先度見直し。
  - 注意点: 媒体別メモを残さず、単一台帳に集約する。
