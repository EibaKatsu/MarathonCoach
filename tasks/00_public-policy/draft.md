# 0. 公開方針の固定

- Source: `TASKS.md`

## T01 [P0] 通常版の提供範囲を1文で定義する

- タスク内容:
  - 通常版の提供範囲を1文で定義する
- 目的:
  - 通常版の価値を一文で固定し、仕様判断の基準を統一するため。
- 草案:
  - 提供範囲の確定文（推奨）: `通常版は、目標タイム達成に必要な判断を1画面に集約し、補給通知とアクション表示でレース中の迷いを減らすGarminデータフィールドを提供する。`
  - 想定ユーザー幅: 初フル完走狙い / サブ4挑戦 / 久しぶりの復帰レース層までを含む。
  - 含める要素: 目標タイム連動 / ACTION表示 / 35分固定補給通知 / ビープ通知。
  - 含めない要素: 個別分析レポート、個別調整、有人サポート付き設定最適化。
  - 注意点: 「誰のどんな不安を減らすか」を1文に必ず含める。
- 結果:
  - `通常版は、目標タイム達成に必要な判断を1画面に集約し、補給通知とアクション表示でレース中の迷いを減らすGarminデータフィールドを提供する。`

## T02 [P0] カスタム版の提供範囲を1文で定義する

- タスク内容:
  - カスタム版の提供範囲を1文で定義する
- 目的:
  - 有料サービスの価値を一文で固定し、通常版との棲み分けを明確にするため。
- 草案:
  - 提供範囲の確定文（推奨）: `カスタム版は、FITデータとヒアリングを基に、あなた専用の設定値・根拠説明・カスタムコードを納品し、狙うタイムに合わせた再現性を高める。`
  - 納品内容に必ず含める: 推奨設定値 / 指標根拠 / 入力手順 / 再調整条件。
  - 通常版との境界: アプリ本体の無料利用と、個別最適化サービスは明確に分離する。
  - 注意点: 「分析します」ではなく「何が手元に残るか（納品物）」を明示する。
- 結果:
  - `カスタム版は、FITデータとヒアリングを基に、あなた専用の設定値・根拠説明・カスタムコードを納品し、狙うタイムに合わせた再現性を高める。`

## T03 [P0] 無料部分と有料部分の境界を確定する

- タスク内容:
  - 無料部分と有料部分の境界を確定する
- 目的:
  - 料金説明と機能説明の不一致を防ぐため。
- 草案:
  - 無料（通常版）: 目標タイム入力 / 距離設定 / レース中表示 / 35分補給通知 / ビープ通知。
  - 有料（カスタム版）: FIT分析レポート / 7項目評価 / 個別設定値 / カスタムコード / 再調整1回。
  - 境界ルール: 「個人データを解釈し、個別提案する作業」は有料。
  - 例外: 不具合修正、公開済み通常版のバグ対応は無料。
  - 訴求文面: `無料でも本番で使える。より自分向けに仕上げる部分が有料。` を固定文として使う。
  - 注意点: 無料・有料の双方に具体例を入れ、問い合わせ時の説明コストを下げる。
- 結果:
  - `無料（通常版）で本番利用に必要な機能を提供し、個人データの解釈と個別提案が必要な最適化部分を有料（カスタム版）として提供する。`

## T04 [P0] 対象ユーザー像を決める

- タスク内容:
  - 対象ユーザー像を決める
- 目的:
  - 誰に最適化するかを固定し、機能追加の判断軸を明確にするため。
- 草案:
  - 主対象: `フルマラソン完走〜サブ4を狙う市民ランナー`。
  - 副対象: ハーフ中心だがフル挑戦を検討している層、復帰レースで失速不安がある層。
  - 課題像: レース中にペース判断・補給判断で迷い、後半失速しやすい。
  - 対象外: トレイル専用運用、医療用途、競歩・自転車向け利用。
  - 訴求軸: 迷い削減 / 補給忘れ防止 / 失速兆候への先回り / 初設定の簡単さ。
  - 注意点: 対象外を先に明記し、期待値ギャップを防ぐ。
- 結果:
  - `主対象はフル完走〜サブ4を狙う市民ランナー、副対象はフル挑戦準備層と復帰層。レース中の迷いと後半失速を減らす価値に集中し、トレイル専用・医療用途・競歩/自転車用途は対象外とする。`

## T05 [P0] 公開時の最小完成条件を決める

- タスク内容:
  - 公開時の最小完成条件を決める
- 目的:
  - 公開延期判断を迷わず実施できる基準を作るため。
- 草案:
  - 必須条件: P0タスク完了 / 致命的不具合0件 / 初回設定2分以内で利用開始可能。
  - 体験条件: 走行中に主要表示が1秒以内で理解できる（テスター評価4段階で上位2段階が80%以上）。
  - 証跡条件: 主要機種帯の実機確認結果、ストア素材、サポート窓口文面、FAQ初版が揃っている。
  - 延期条件: 走行中の誤誘導リスクがある通知不具合、設定保存不可、再現性のあるクラッシュ。
  - 注意点: 「できれば」を排除し、公開可否を二択で判定できる形式にする。
- 結果:
  - `公開条件は、P0完了・致命的不具合0件・初回設定2分以内・主要表示の視認性基準達成・証跡資料一式の整備完了とする。未達が1つでもあれば公開延期。`
