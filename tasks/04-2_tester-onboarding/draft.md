## T01 [P0] テスター応募フォームを作る

- タスク内容:
  - テスター応募フォームを作る
- 目的:
  - 選考・機種配分に必要な情報を漏れなく取得するため。
- 草案:
  - 必須項目: ハンドル名 / 連絡先 / 使用機種 / 10km以上検証可否。
  - 任意項目: ラン歴 / 主なレース距離 / 利用中デバイス / 直近目標タイム。
  - 同意項目: テスト結果の匿名利用可否、連絡目的での情報利用同意。
  - 回答設計: 自由記述を減らし、選択式中心で集計しやすくする。
  - 注意点: 最終確認画面で応募後の流れ（配布日・返信期限）を再掲する。

## T02 [P0] ベータテスターを5〜10人集める

- タスク内容:
  - ベータテスターを5〜10人集める
- 目的:
  - 募集導線を実際の応募数へつなげ、主要機種帯の検証体制を作るため。
- 草案:
  - 募集目標: 合計8名（下限5・上限10）。
  - 機種配分目標: Forerunner系3名 / Venu-vivoactive系3名 / fenix-epix系2名。
  - 達成条件: 各機種帯で1件以上の走行フィードバックを取得。
  - 注意点: 人数だけでなく機種分散を達成条件に含め、募集期間中に不足帯を補正する。

## T03 [P0] テスター向け案内テンプレートを作る

- タスク内容:
  - テスター向け案内テンプレートを作る
- 目的:
  - 配布後の初動を揃え、検証品質を均一化するため。
- 草案:
  - 案内テンプレ本文: インストール手順 / 設定値例 / テスト観点 / 返信方法。
  - 依頼観点: 初回設定で迷った点、見やすさ、通知の多さ、不具合。
  - 初動導線: `まず15分走で確認してほしい項目` を先頭に提示する。
  - 返信期限: 配布後7日以内、形式はフォームまたは返信テンプレ。
  - 注意点: 最初の1通で完結するように必要URLを全て含める。

## T04 [P0] フィードバック回収項目を固定する

- タスク内容:
  - フィードバック回収項目を固定する
- 目的:
  - 機種間比較ができる形でフィードバックを収集するため。
- 草案:
  - 固定項目: 機種 / インストール可否 / 初回設定迷い / 視認性 / 通知頻度。
  - 不具合項目: 事象 / 発生タイミング / 再現性 / 画面写真。
  - 継続意向: `継続利用したいか` を3段階で回収。
  - 推奨度項目: `ラン仲間に勧めたいか` を5段階で回収し、訴求文改善に活用。
  - 注意点: 項目を途中で変えない。変更時は版番号を付与する。

## T05 [P1] テスターへのお礼方針を決める

- タスク内容:
  - テスターへのお礼方針を決める
- 目的:
  - 協力依頼の透明性を高めるため。
- 草案:
  - 基本方針: 無償協力を前提、希望者に謝辞掲載。
  - 任意特典: カスタム版の初回割引クーポン（公開後1か月有効）。
  - 非金銭特典: `先行アップデート案内` と `テスター限定改善レポート共有`。
  - 告知文: `金銭報酬はありません。謝辞掲載と先行情報提供を行います。`
  - 注意点: 募集時点でお礼内容を明示し、期待差をなくす。
