# 1-1. 機能凍結と仕様確定

- Source: `TASKS.md`

## T01 [P0] 公開版で入れる機能を最終確定する

- タスク内容:
  - 公開版で入れる機能を最終確定する
- 目的:
  - 公開版の実装範囲を固定し、追加開発を防ぐため。
- 草案:
  - 公開版機能: 距離進捗表示 / 目標タイム連動ペース表示 / ACTION表示 / 35分固定補給通知 / ビープ通知。
  - 体験コンセプト: `走行中3秒で次の行動が決まる` を最上位要件にする。
  - 設定項目: `raceDistanceKm` と `targetTimeSec` の2項目のみ。
  - ストア訴求に直結する言い換え: `ペースを見る` ではなく `次に何をするかが分かる`。
  - 注意点: 公開版機能はPRODUCT.mdのスコープと一致させる。
- 結果:
  - `公開版は、3秒判断を支える5機能（距離進捗/目標連動ペース/ACTION/35分補給通知/ビープ通知）に固定し、設定は2項目に限定する。`

## T02 [P0] 公開版で入れない機能を明文化する

- タスク内容:
  - 公開版で入れない機能を明文化する
- 目的:
  - ユーザー期待値と実装計画のズレを防ぐため。
- 草案:
  - 非搭載機能: FIT自動取り込み、詳細なレース後分析、公開版の設定項目拡張。
  - 非搭載理由: 公開初期は走行中の安定表示と通知品質を優先するため。
  - 期待値調整文: `公開版はレース中支援に特化。レース後の深掘りは今後の検討またはカスタム版で提供。`
  - 代替導線: 非搭載項目に関心が高いユーザー向けに、カスタム版案内リンクを説明文末尾へ配置。
  - 注意点: 将来検討と公開版提供を混同しない表現に統一する。
- 結果:
  - `公開版非搭載機能とその理由、代替導線（カスタム版案内）を明文化し、期待値ギャップを防ぐ文面へ統一する。`

## T03 [P0] 通常版の設定項目を最終確定する

- タスク内容:
  - 通常版の設定項目を最終確定する
- 目的:
  - 初回設定の迷いを最小化するため。
- 草案:
  - 設定項目1: `raceDistanceKm`（初期値 `42.195`、許容範囲 `5.0-60.0`）。
  - 設定項目2: `targetTimeSec`（初期値 `05:00:00`、許容範囲 `02:30:00-07:00:00`）。
  - 固定仕様: 通知有効時の補給間隔は35分固定、通常版では変更不可。
  - 距離別補給仕様:
    - フル（上記以外）: 35分ごとに補給通知。
    - ハーフ（`20.8-21.4km`）: 35分通知を1回のみ。
    - 10K（`9.5-10.5km`）: 補給通知なし。
    - 5K（`4.5-5.5km`）: 補給通知なし。
  - 入力異常時: 許容外の値は保存せず初期値に戻す。
  - 導入体験KPI: 初回設定完了まで60秒以内を目標にする。
  - 注意点: 設定を増やさず、公開版の説明負荷を抑える。
- 結果:
  - `設定は距離と目標タイムの2項目に固定する。補給は距離別仕様（フル:35分ごと、ハーフ:35分1回、10K/5K:通知なし）を採用し、入力異常は保存せず安全側へフォールバックする。`

## T04 [P0] エラー時の挙動を決める

- タスク内容:
  - エラー時の挙動を決める
- 目的:
  - 異常値時に危険な通知を出さないため。
- 草案:
  - 心拍取得不可: 心拍依存の警告は停止し、画面に `HR取得なし` を表示。
  - GPS不安定: 距離・ペースが急変した場合は直前安定値を最大10秒保持。
  - 設定未入力: 初期値で起動し、開始前に設定確認メッセージを1回表示。
  - 異常値入力: 保存時に弾き、初期値にフォールバック。
  - 信頼維持文言: `不確かな時は通知しない` をエラー設計の原則として仕様に明記。
  - 注意点: 安全側に倒すフォールバックを優先し、過剰通知を抑える。
- 結果:
  - `エラー時は安全側フォールバックを最優先し、「不確かな時は通知しない」原則で心拍/GPS/設定異常時の挙動を固定する。`
