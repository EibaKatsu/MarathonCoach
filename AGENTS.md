## Product Value (Runner-Centric)
このアプリは、レース中の意思決定を減らし、完走・目標達成確率を上げる「リアルタイム伴走」を提供する。

### Core runner value
- ペース迷子を防ぐ: 目標との差分と Push/Hold/Ease で、次の一手を即時に示す。
- 失速リスクを下げる: 心拍ゾーン超過を早期検知し、オーバーペース修正を促す。
- 補給忘れを防ぐ: 補給タイマーと警告で後半のガス欠リスクを抑える。
- 崩れの兆候に先回りする: ドリフト兆候に対して水分/補給などの対処を促す。
- メンタル負荷を下げる: 短いメッセージで焦りを抑え、リズム維持に集中させる。

### Guidance for future feature proposals
- 単なるデータ表示より「次の行動提案」に変換できる機能を優先する。
- 走行中に迷わないことを最優先し、表示は短く、判断は即時にする。
- 後半失速を防ぐ観点 (ペース/心拍/補給/ドリフト) への寄与を明示する。
- 新機能は「完走確率向上」または「目標達成確率向上」のどちらに効くかを説明する。

## 距離通知（現在実装）
距離通知カードは **3行固定フォーマット**。
- 1行目：距離ラベル（例 `8km` / `HALF` / `FULL`）
- 2行目：意味づけ
- 3行目：短い行動メッセージ

### 種目別ルール
- フルマラソン：
  - 1kmごとに通常スプリット
  - `21.1km` で節目カード `HALF`
  - `42.2km` でゴールカード `FULL`
- ハーフマラソン：
  - 1kmごとに通常スプリット
  - `10km` で節目カード `10km`
  - `21.1km` でゴールカード `HALF`
- 10km：
  - 1kmごとに通常スプリット
  - `5km` で節目カード `5km`
  - `10km` でゴールカード `10km`
- 5km：
  - 1kmごとに通常スプリット
  - 中間節目カードなし
  - `5km` でゴールカード `5km`

### 通常スプリット文言
- 序盤 / 中盤 / 後半の3距離帯テンプレートを使用
- 各距離帯に3セットを持ち、`(km - 1) % 3` でローテーション

### 判定と表示
- 到達判定：`distance >= target - 0.02km`
- 表示時間：**7秒**
- 表示優先順位：
  1. `FUEL NOW`
  2. `HR OVER`
  3. `DRIFT ON`
  4. 距離通知カード
  5. 通常カード（ACTION/FUEL）
- 警告カード表示中に到達した距離通知はスキップする

## カスタムモード（MVP方針・再構成版）

### 目的
カスタムモードは、通常版では扱わないランナー個人の特性を反映し、補給・心拍管理・レース運びを個別最適化するためのモードとする。

通常版は「距離」「目標タイム」を中心としたシンプル設計を維持する。  
カスタムモードでは、意味が伝わりやすく、体感差が出やすく、実装負荷が過度に高くならない項目に限定して個別設定を許可する。

### MVPで採用する設定項目
カスタムモードの初期実装では、設定項目を次の7つに限定する。

1. **補給方式**  
   - 内部キー例: `fuelMode`
   - 内容: 補給通知を使うかどうか
   - 値の例: `off` / `time`
   - 方針: MVPでは「補給なし」と「時間ベース補給通知」の2択に限定する

2. **初回補給タイミング**  
   - 内部キー例: `firstFuelAfterMin`
   - 内容: レース開始後、最初の補給通知を何分後に出すか
   - 目的: 「最初だけ早めに入れたい」などの個人差に対応する
   - 値の例: 10〜30分、1分刻み
   - 備考: `fuelMode=time` の場合のみ有効

3. **補給間隔**  
   - 内部キー例: `fuelIntervalMin`
   - 内容: 初回補給後、何分ごとに補給通知を繰り返すか
   - 目的: ジェル摂取頻度の個人差に対応する
   - 値の例: 15〜45分、1分刻み
   - 備考: `fuelMode=time` の場合のみ有効

4. **補給予告の早さ**  
   - 内部キー例: `fuelAlertLeadMin`
   - 内容: 補給予定の何分前から予告表示を出すか
   - 目的: 補給準備のしやすさを調整する
   - 値の例: 0〜5分、1分刻み
   - 備考: `fuelMode=time` の場合のみ有効

5. **走り方強度**  
   - 内部キー例: `phaseAggressiveness`
   - 内容: レース全体をどれくらい守り寄り / 攻め寄りで運ぶか
   - 目的: 「かなり攻めたい」などの強い個別要望に対応する
   - 値の例: 0〜20 の多段階スケール
   - 目安:
     - 0 = 超守り
     - 10 = 標準
     - 20 = かなり攻め
   - 備考: 内部ではフェーズごとの許容度、終盤の押し上げやすさなど複数の閾値に反映する

6. **心拍上限バイアス**  
   - 内部キー例: `hrCapBiasBpm`
   - 内容: 標準設定から何bpm安全側または攻め側に寄せるか
   - 目的: 心拍が上がりやすい人、暑さに弱い人、終盤失速しやすい人に対応する
   - 値の例: -8〜+8 bpm、1bpm刻み
   - 備考: マイナスは安全側、プラスは攻め側

7. **ドリフト感度**  
   - 内部キー例: `driftSensitivity`
   - 内容: 心拍ドリフトや疲労兆候の警告をどれだけ敏感に出すか
   - 目的: 警告の出方を個人に合わせる
   - 値の例: 0〜7 の多段階スケール
   - 目安:
     - 0 = 鈍感
     - 3 = 標準
     - 7 = 敏感

### 補給設定のMVP方針
補給系設定は複雑にしすぎず、MVPでは `fuelMode` を次の2択に限定する。

- `off` : 補給通知なし
- `time` : 時間ベースで補給通知を出す

マニュアル補給はアプリ設定としては扱わない。  
ユーザーが自分の判断で補給する運用は許容するが、通知ロジックとしては持たない。

`fuelMode=time` の場合のみ、次の設定を有効にする。

- `firstFuelAfterMin`
- `fuelIntervalMin`
- `fuelAlertLeadMin`

挙動は次の通りとする。

- `fuelMode=off` の場合、補給通知は一切出さない
- `fuelMode=time` の場合、`firstFuelAfterMin` で最初の補給通知を出す
- 2回目以降は `fuelIntervalMin` ごとに補給通知を繰り返す
- 補給予定時刻の `fuelAlertLeadMin` 分前から予告表示を出す

将来的に距離ベース補給が必要になった場合は、`fuelMode` に `distance` を追加して拡張する。  
MVP段階では対象外とし、仕様の単純さを優先する。

### 設計方針
- カスタムモードでユーザーが直接触る項目は、上記7項目に絞る
- 数値系設定は固定プリセット番号ではなく、範囲付きの数値や多段階スケールを整数化して扱う
- これによりコード長を抑えつつ、「かなり攻めたい」などの強い個別要望にも対応できるようにする
- 専門性が高く説明コストの高い閾値は、ユーザーに直接見せず内部パラメタとして管理する
- `phaseAggressiveness` や `driftSensitivity` のような抽象設定は、内部で複数の制御値へ展開する
- 通常版はシンプルさを優先し、カスタムモードのみ個別最適化を許可する
- カスタムコード方式を採用する場合も、外部に見せる論理項目はこの7つを基本単位とする

### カスタムコード方針
カスタムモードでは、ユーザー個別設定を毎回ビルドで埋め込むのではなく、上記7項目の設定値を短いカスタムコードへ符号化して受け渡す方式を基本とする。  
アプリ側はカスタムコードを展開して設定値へ反映する。

コードには少なくとも次を含める。

- バージョン情報
- 設定本体
- 誤入力検知用チェック値

このコードは厳密な秘密保持用の暗号ではなく、運用性・入力性・拡張性を重視した設定表現として扱う。

### 初期実装でユーザーに見せない内部パラメタ
初期段階では、次のような項目は直接公開しない。  
必要があれば `phaseAggressiveness` や `driftSensitivity` から内部的に自動決定する。

- フェーズ境界の細かな進行率
- 心拍オーバー判定秒数
- 心拍オーバー解除秒数
- 心拍オーバー解除用オフセットbpm
- ペース押し上げ判定秒差
- push時の心拍余裕幅
- cardiac cost 関連の細かな比率
- 通知固定表示秒数
- 誤操作防止用のデバウンス秒数

### この設計の狙い
- 通常版との差を明確にする
- ユーザーにとって意味が伝わる設定だけを表に出す
- 実装複雑度を抑えつつ、個別最適化の価値を出す
- 将来的な拡張時も、表向きの設定体系を崩さずに内部制御のみ強化できるようにする

## 補給判定の整合ルール（カード / 右上メーター）
- 補給間隔は `35:00` 固定。`Lap` 押下で `35:00` へリセットする。
- 残り `> 2:00`：メーターは **通常**、カードは通常表示（FUELトグル対象外）。
- 残り `2:00〜0:01`：メーターは **注意**、カードは `ACTION/FUEL` を3秒トグルする。
- 残り `<= 0:00`（期限到達/超過、Lapまで継続）：メーターは **警告**、カードは **FUEL NOW固定（最優先）**。
- 右上メーター警告文言は固定で `補給now / 補給後LAP`（英語: `Fuel now / Press LAP`）。

## 優先順位（表示固定ルール）
最優先から順に適用し、該当すればコーチカードを固定する。

1. **FUEL NOW（期限超過）**
   - カード：`いま 補給`
2. **HR > ZONE上限（心拍上限超過）**
   - カード：`ちょい落とし`
3. **DRIFT ON（ドリフト検知）**
   - カード：`水＋補給`
4. 通常
   - トグル表示（ACTION ↔ FUEL）

## レースプロファイル判定（現在実装）
`raceDistanceKm` の設定値でプロファイルを決める。

- `SHORT`：`raceDistanceKm <= 10.5`（5km/10km想定）
- `HALF`：`abs(raceDistanceKm - 21.0975) <= 0.25`
- `FULL`：上記以外（フルマラソン想定）

## ZONE（心拍上限）ロジック（Garmin心拍ゾーン基準）
距離の絶対値ではなく、`progress = elapsedDistance / raceDistanceKm` の進捗率で5フェーズに分割する。

- `S1`: `< 24%`
- `S2`: `24%〜59%`
- `S3`: `59%〜83%`
- `S4`: `83%〜95%`
- `S5`: `95%〜Finish`

許容ゾーン（`allowedZoneNumber`）とオフセット（`allowedZoneOffsetBpm`）：

| Profile | S1 | S2 | S3 | S4 | S5 |
| --- | --- | --- | --- | --- | --- |
| FULL | Z2 +0 | Z3 +0 | Z3 +2 | Z4 +0 | Z4 +3 |
| HALF | Z3 +0 | Z4 -2 | Z4 +0 | Z4 +2 | Z4 +4 |
| SHORT | Z4 +0 | Z5 +2 | Z5 +3 | Z5 +4 | Z5 +5 |

HR超過判定（`HR > allowed + 1bpm`）：
- Over開始継続秒数：
  - `S1/S2/S3`: `12秒`
  - `S4`: `10秒`
  - `S5`: `20秒`
- Over解除：
  - 回復継続 `5秒`
  - 解除閾値は `S4` のみ `allowed -1bpm`、それ以外は `allowed -2bpm`

## ACTION判定（現在実装）
「ちょい上げ」は、遅れがあり、かつ心拍余裕があり、Cardiac Costが悪化していない場合のみ発火する。

- フェーズ定義は ZONE と同じ（`S1..S5`）。
- Cardiac Cost比：
  - `baseCost = baseHR * basePace`
  - `curCost = curHR * curPace`
  - `ccRatio = curCost / baseCost`
  - `ccRatio` は次の条件を満たす場合のみ有効（それ以外は `null` 扱い）：
    - ドリフト基準確定済み（`baseHR/basePace` が有効）
    - rollingサンプル数 `>= 30`
    - `abs(curPace - basePace) <= 5秒/km`（pace stability 条件）

- 前提条件（1つでも満たさない場合は発火しない）：
  - FUEL期限超過でない
  - HR超過状態でない
  - DRIFT ONでない
  - `paceNowSecPerKm` / `targetPaceSecPerKm` / `allowedMaxHeartRate` / `currentHeartRate` が有効
- ちょい上げ発火閾値（`paceDelta >=` かつ `headroom >=`）：

| Profile | S1 | S2 | S3 | S4 | S5 |
| --- | --- | --- | --- | --- | --- |
| FULL | `+12s, 8bpm` | `+8s, 6bpm` | `+6s, 4bpm` | `+4s, 3bpm` | `+3s, 2bpm` |
| HALF | `+10s, 7bpm` | `+6s, 5bpm` | `+4s, 3bpm` | `+3s, 2bpm` | `+2s, 1bpm` |
| SHORT | `+8s, 6bpm` | `+5s, 4bpm` | `+3s, 2bpm` | `+2s, 1bpm` | `+1s, 0bpm` |

- Cardiac Costゲート（ちょい上げ時）：
  - `FULL`: `ccRatio <= 1.06`
  - `HALF`: `ccRatio <= 1.08`
  - `SHORT`: `ccRatio <= 1.10`
  - `ccRatio == null` の場合はゲート未適用（発火可）
- ヒステリシス：
  - ON：条件成立が `6秒` 継続
  - OFF：`paceDelta < (閾値 -2秒)` または `headroom < (閾値 -1bpm)` または Cardiac Costゲート不成立が `5秒` 継続
- ちょい落とし（Ease）判定：
  - `paceDelta <= -8秒/km` または `headroom <= 閾値` または `baselineHrDelta >= 閾値` または `ccRatio >= 閾値`
  - `headroom` / `baselineHrDelta` / `ccRatio` の閾値はプロファイル・フェーズ別に可変
    - `ccRatio` 閾値：`FULL 1.10 / HALF 1.12 / SHORT 1.15`

### ドリフトON/OFF
- ON：`curHR - baseHR >= +10 bpm`
- OFF：`curHR - baseHR <= +6 bpm` を **60秒継続**

### UI
- ドリフトON時：コーチカードを **「水＋補給」固定**（トグル停止）

## STEP3レイアウト方針（確定）
### 対象デバイス
- 基準：`fr255`
- デバイス差分は機種個別分岐ではなく、`small / medium / large` のサイズクラスで吸収する
  - `small`: `min(width, height) <= 218`
  - `medium`: `219 .. 260`（`fr255`はここ）
  - `large`: `>= 261`

### STEP3で固定する範囲
- 固定対象は **外枠（safe area）+ 内部分割（中央割 + 縦4分割）**
- ロジック（判定、トグル、固定表示の条件）はSTEP4以降で実装する

### レイアウトモデル（中央割 + 縦4分割）
- 分割ルール：画面を中央で左右に分け、さらに上下を4段に分ける（計8セル）
- 添付モック（`assets/step3_layout_fr255_4block_mockup.png`）準拠で配置する
- 要素配置：
1. 1段目（上段）：左=`HR/ZONE`、右=`FUELリング上部（右1-2段跨ぎ）`
2. 2段目：左=`CARD（上半）`、右=`FUELリング下部（右1-2段跨ぎ）`
3. 3段目：左=`CARD（下半）`、右=`PACE（大） + /km`
4. 4段目（下段）：左=`DIST`、右=`TIME`（`PACE Δ` は下段内で補助表示）

### 座標・サイズの決め方
- 画面絶対値ではなく **比率ベース** を使う
- 主要寸法は `ratio + min/max px` のガードで定義する
  - 例：`outerPadding = 5%`（`min 8px / max 16px`）
- ブロック間ギャップも比率で定義し、クラス別に微調整する

### 表示優先順位（狭小時）
- 欠けを避ける優先順は以下とする
1. コーチカード文言（CARD）
2. 現在ペース（PACE）
3. HR/ZONE（1段目・左セル）
4. FUEL残り時間（右1-2段跨ぎセル）
5. DIST/TIME（4段目・左右セル）
6. PACE Δ（最初に縮退/省略対象）
