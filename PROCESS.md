# PROCESS

## 修正時の実行順（必須）
1. 修正
2. テスト
3. PR作成
4. 独立レビュー
5. マージ

## 実行ポリシー
- デフォルト: 上記 1〜5 をすべて実施
- 例外: ユーザーが「ローカルのみ」と明示した場合のみ 3〜5 を省略可
- 完了条件: 5. マージまで完了していない作業は「完了」としない

## 最終報告の必須項目
- `PR URL`
- `独立レビュー結果`（指摘有無、主な指摘、対応状況）
- `マージコミット`（main に入った commit SHA）

## 各ステップの目的
1. 修正: 要件を実コードへ反映
2. テスト: 回帰防止と動作確認
3. PR作成: 変更意図・影響範囲の共有
4. 独立レビュー: 品質課題の事前検出
5. マージ: 品質担保済み変更の正式反映

## GitHub ブランチ保護
- `Require a pull request before merging`
- `Require approvals`

## 診断ログ方針
不具合が簡単に解決できない場合は、対象ロジック周辺に診断ログを追加し、
`入力値 / 判定結果 / 表示値` を実測してから修正する。

- 原因特定後は `DEBUG` フラグで無効化、または不要ログを削除する
