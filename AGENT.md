# Marathon Coach Data Field (Connect IQ) - AGENT.md

## 目的
市民マラソン向けの **Connect IQ データフィールド（1画面）** を実装する。  
通常の「数値の羅列」ではなく、**数値から導かれる“次のアクション”をコーチカードとして提示**する。

- 主対象：PB 5:00前後のランナー
- レース中は判断負荷を最小化し、事故（オーバーペース／補給遅れ／脱水兆候）を減らす

## 現在スコープ（暫定）
- 当面は **FREEモード（Core）を完璧に仕上げることを最優先** とする。
- `FREE / PRO / CUSTOM` のモード定義・フラグ実装は **一旦スコープ外** とする。
- モード定義は **TBD**（仕様確定後に再開）とする。

---

## 成果物
- Connect IQ Data Field（1面）
- 主要ロジック：
  - 心拍上限（Garmin心拍ゾーン基準）
  - 補給タイマー（LAPでリセット）
  - 補給間隔35分固定（無料版/PRO版）
  - カスタム版のみ補給間隔を個別調整
  - ドリフト検知（基準区間20–30分）
  - ACTION/FUELの交互表示（ただし特定状態では固定）

## UI方針（コーチカード風 / B案）
- 画面中央に「コーチカード」を表示し、**短い文言**で行動指示を出す（日本語/英語対応）
- 数値は補助。視線の主役は「カード」
- **右下：いまのぺーす**（10秒移動平均）
  - 数字は上寄せ、単位 `/km` は下に小さく

### 画面内の要素（1画面）
- 中央：コーチカード（状態に応じて文言・色・アイコン）
- 左上付近：心拍（HR）とゾーン（ZONE）
  - Garmin心拍ゾーン取得時：`HR/Zn`（例 `152 / Z3`）を表示
  - ゾーン取得不可時：`HR / --`
- 右上：FUELリング＋残り時間
- 右下：いまのぺーす（10秒平均）＋ `/km`（小）
- 下段：距離（DIST）／経過時間（TIME）
- 目標との差（PACE Δ）：表示（ただし数値羅列感を抑え、短く）

## 無料版 / PRO版 / カスタム版（切り分け）※TBD（現時点はスコープ外）
本アプリは **無料版（Core）** を基盤にし、PRO版は **通知（A）＋後半レースモード（C）** を追加する。  
カスタム版は PRO相当の機能を前提に、個別パラメータを設計して配布し、必要に応じてサポートを付与する。

本章と他章に矛盾がある場合は、**本章（追加版）を優先**する。

補給間隔は **無料版・PRO版は35分固定**、調整は **カスタム版のみ**とする。

### 無料版（Core）
目的：信頼できる“土台”を提供し、評価を落とさない（判定のバタつきを抑える）。

含まれるもの：
- コーチカード（ちょい上げ／ちょい落とし／そのまま／ほきゅう系）
- 右下：いまのぺーす（10秒平均、単位は下に小さく）
- 距離／経過時間
- 心拍ZONE上限（Garmin心拍ゾーン基準、距離帯で段階）
- ドリフト検知（基準20:00〜30:00、10分窓）＋ ドリフトONで「みず＋ほきゅう」固定
- 補給タイマー（ACTION/FUEL交互表示）＋ LAPで補給リセット（案B）
- 補給間隔：**35分固定**
  - LAPでリセットした時点から次の35分を再カウント
- **判定の安定化（B）**：
  - ペース判定の平滑化（表示10秒平均と、判定用の平滑化を分ける）
  - HR>ZONE上限／ドリフトON/OFFのバタつき防止（ヒステリシス、継続秒数、クールダウン）
  - GPS不安定時はペース判定の重みを下げ、心拍・補給を優先する

無料版に含めない（PRO差分として残す）：
- バイブ等の通知強化
- kmごとのラップ通知
- 後半レースモード（30km以降の意思決定強化）
- 補給間隔の調整

---

### PRO版（IAP：通知A + 後半モードC）
目的：レース当日の取りこぼしを減らし、後半（30km〜）の意思決定を強くする。

追加されるもの（無料版＋以下）：
- **通知強化（A）**
  - 重要状態のバイブ通知：
    - いま ほきゅう（補給期限超過）
    - ちょい落とし（HR>ZONE上限）
    - みず＋ほきゅう（ドリフトON）
  - 重要状態の表示固定（一定時間強制表示）や、通知の間引き（実装方針に従う）

- **kmごとのラップ通知（推奨仕様）**
  - 1kmごとに通知（頻度はPROで設定可能にしても良い）
  - 通知文言は 1行完結 を基本とし、入り切らない場合は自動短縮する
    - 通知テンプレ（1行）：
      - `{km}km {pace} {delta} {action}`
      - 例：`12km 7:08 +2 そのまま`
      - 例：`28km 7:18 +12 みず＋ほきゅう`
    - 自動短縮の優先度（削る順）：
      - 差分 → ペース → km（指示は必ず残す）
      - 最小形：`そのまま` / `みず＋ほきゅう` / `ちょい落とし` / `いま ほきゅう`
  - 通知の優先順位（競合時）：
    1) いま ほきゅう
    2) ちょい落とし（HR>ZONE上限）
    3) みず＋ほきゅう（DRIFT ON）
    4) kmラップ通知

- **後半レースモード（C）**
  - 30km以降は「ちょい上げ」を基本抑制し、条件が揃った場合のみ許可する
  - 条件例（実装で調整可）：
    - HRがZONE上限内で余裕がある
    - DRIFT OFF
    - 直近で補給済み（LAPから一定時間以内）
    - ペースが安定（GPSが荒れていない）
  - 崩れ方に応じて指示を変える（例）：
    - 心拍↑＋ペース↓：みず＋ほきゅう寄り＋ちょい落とし
    - 心拍↔/↓＋ペース↓：そのまま寄り（脚終わり系）

- 補給間隔：**35分固定**（無料版と同様。調整UIは持たない）

### カスタム版（個別最適化 + サポート）
目的：購入者の特性（心拍ゾーン設定、目標、暑さ耐性、胃、補給能力、失速癖）に合わせて、PRO機能のパラメータを“設計”して提供する。

含まれるもの：
- 機能は PRO相当 を前提（通知、kmラップ、後半モードを含む）
- 個別最適化の対象（例）：
  - **補給間隔の調整**（30–40分レンジ、前倒し幅、距離ゲート等）
  - ZONE段階の差分（守り/攻め）
  - ドリフト閾値、解除条件、クールダウン
  - 後半モードのPUSH許可条件（厳しさ）
  - 通知の強さ、頻度、固定表示時間
- 配布方法：
  - **カスタムコード**（1つのコード入力で一括反映）を採用する
  - 受領後、ユーザーは設定画面にコードを入力して反映（詳細は別ドキュメント化）

サポート（任意）：
- ヒアリング → 設定/当日プラン納品 → 微調整 → レース後レビュー（商品メニューに応じる）

## ユーザー設定（Settingsラベルは英語固定）
ユーザーが変更できる設定は **2つのみ**（最大心拍は出さない）。

1. **レース距離**（km）
   - default: 42.195
2. **目標タイム**（HH:MM:SS）
   - default: 05:00:00

補給間隔は無料版/PRO版では **35分固定**（ユーザー設定なし）。
調整はカスタム版のみ。

心拍ゾーンは **Garmin端末設定値を取得する前提**（設定項目なし）。

## データ定義（内部）
- `raceDistanceKm`（設定）
- `targetTimeSec`（設定）
- `targetPaceSecPerKm` = `targetTimeSec / raceDistanceKm`
- `fuelIntervalSec`（無料版/PRO版は固定 `35 * 60`。カスタム版のみ個別値）
- `lastFuelTimeSec`（LAP押下で更新）
- `fuelDueTimeSec` = `lastFuelTimeSec + fuelIntervalSec`

### ペース（表示用）
- `paceNowSecPerKm`：**10秒移動平均**
- `paceDeltaSecPerKm` = `paceNowSecPerKm - targetPaceSecPerKm`

## 入力（LAP操作）
### 補給完了リセット（確定：案B）
- ランナーが補給できたタイミングで **LAPボタン押下**
- 押下時点を「補給完了」とし、以降の `fuelDueTimeSec` を再計算
- 補給が遅れても、**リセットした時点から改めてカウント開始**する

誤操作対策：
- `lapDebounceSec`（例：20秒）以内の連続リセット無効

- 通常：**3秒ごとに交互表示**
  - ACTION ↔ FUEL
- ただし、以下の状態では固定（トグル停止）

## 補給判定の整合ルール（カード / 右上メーター）
- 補給間隔は `35:00` 固定。`Lap` 押下で `35:00` へリセットする。
- 残り `> 2:00`：メーターは **通常**、カードは通常表示（FUELトグル対象外）。
- 残り `2:00〜0:01`：メーターは **注意**、カードは `ACTION/FUEL` を3秒トグルする。
- 残り `<= 0:00`（期限到達/超過、Lapまで継続）：メーターは **警告**、カードは **FUEL NOW固定（最優先）**。
- 右上メーター警告文言は固定で `補給now / 補給後LAP`（英語: `Fuel now / Press LAP`）。

---

## 優先順位（表示固定ルール）
最優先から順に適用し、該当すればコーチカードを固定する。

1. **FUEL NOW（期限超過）**
   - カード：`いま 補給`
2. **HR > ZONE上限（心拍上限超過）**
   - カード：`ちょい落とし`
3. **DRIFT ON（ドリフト検知）**
   - カード：`水＋補給`
4. 通常
   - トグル表示（ACTION ↔ FUEL）

---

## ZONE（心拍上限）ロジック（Garmin心拍ゾーン基準）
Garminの心拍ゾーンを取得できる場合、距離帯で許容上限を切り替える。

- 0–10km：`Z2上限`
- 10–25km：`Z3上限`
- 25–35km：`Z3上限 +2bpm`
- 35–40km：`Z4上限`
- 40km–Finish：`Z4上限 +3bpm`（警告は弱め）

HR超過判定はバタつき防止のため、
- Over開始：上限`+1bpm`を継続
  - 0–35km：`12秒`
  - 35–40km：`10秒`
  - 40km–Finish：`20秒`
- Over解除：回復閾値を `5秒` 継続
  - 0–35km / 40km–Finish：`HR <= 許容上限 -2bpm`
  - 35–40km：`HR <= 許容上限 -1bpm`

---

## ちょい上げ判定（現在実装）
「ちょい上げ」は、遅れがあり、かつ心拍余裕がある場合にのみ発火する。

- 前提条件（1つでも満たさない場合は発火しない）：
  - FUEL期限超過でない
  - HR超過状態でない
  - DRIFT ONでない
  - `paceNowSecPerKm` / `targetPaceSecPerKm` / `allowedMaxHeartRate` / `currentHeartRate` が有効
- 距離帯ごとの発火閾値：
  - 0–10km：`paceDelta >= +12秒/km` かつ `headroom >= 8bpm`
  - 10–25km：`paceDelta >= +8秒/km` かつ `headroom >= 6bpm`
  - 25–35km：`paceDelta >= +6秒/km` かつ `headroom >= 4bpm`
  - 35–40km：`paceDelta >= +4秒/km` かつ `headroom >= 3bpm`
  - 40km–Finish：`paceDelta >= +3秒/km` かつ `headroom >= 2bpm`
- ヒステリシス：
  - ON：条件成立が `6秒` 継続
  - OFF：`paceDelta < (閾値 -2秒)` または `headroom < (閾値 -1bpm)` が `5秒` 継続

## ドリフト検知（確定：方式B）
### 基準区間（Baseline）
- レース開始後、**20:00経過 かつ 3.0km到達** を満たした時点から連続10分で基準を作る
  - `basePace`（平均ペース）
  - `baseHR`（平均心拍）
- 基準確定前（ACTION判定に入る前）は、コーチカードにウォームアップ文言を表示する
  - 文言はローテーション表示し、切替間隔は **15秒**
  - 基準確定後は通常のACTION判定文言（ちょい上げ / そのまま / ちょい落とし）へ切り替える

### 判定窓（Rolling）
- 直近10分：
  - `curPace`（平均ペース）
  - `curHR`（平均心拍）

### ペース安定条件
- `abs(curPace - basePace) <= 5秒/km` の場合のみ判定する

### ドリフトON/OFF
- ON：`curHR - baseHR >= +10 bpm`
- OFF：`curHR - baseHR <= +6 bpm` を **60秒継続**

### UI
- ドリフトON時：コーチカードを **「水＋補給」固定**（トグル停止）

## STEP3レイアウト方針（確定）
### 対象デバイス
- 基準：`fr255`
- デバイス差分は機種個別分岐ではなく、`small / medium / large` のサイズクラスで吸収する
  - `small`: `min(width, height) <= 218`
  - `medium`: `219 .. 260`（`fr255`はここ）
  - `large`: `>= 261`

### STEP3で固定する範囲
- 固定対象は **外枠（safe area）+ 内部分割（中央割 + 縦4分割）**
- ロジック（判定、トグル、固定表示の条件）はSTEP4以降で実装する

### レイアウトモデル（中央割 + 縦4分割）
- 分割ルール：画面を中央で左右に分け、さらに上下を4段に分ける（計8セル）
- 添付モック（`assets/step3_layout_fr255_4block_mockup.png`）準拠で配置する
- 要素配置：
1. 1段目（上段）：左=`HR/ZONE`、右=`FUELリング上部（右1-2段跨ぎ）`
2. 2段目：左=`CARD（上半）`、右=`FUELリング下部（右1-2段跨ぎ）`
3. 3段目：左=`CARD（下半）`、右=`PACE（大） + /km`
4. 4段目（下段）：左=`DIST`、右=`TIME`（`PACE Δ` は下段内で補助表示）

### 座標・サイズの決め方
- 画面絶対値ではなく **比率ベース** を使う
- 主要寸法は `ratio + min/max px` のガードで定義する
  - 例：`outerPadding = 5%`（`min 8px / max 16px`）
- ブロック間ギャップも比率で定義し、クラス別に微調整する

### 表示優先順位（狭小時）
- 欠けを避ける優先順は以下とする
1. コーチカード文言（CARD）
2. 現在ペース（PACE）
3. HR/ZONE（1段目・左セル）
4. FUEL残り時間（右1-2段跨ぎセル）
5. DIST/TIME（4段目・左右セル）
6. PACE Δ（最初に縮退/省略対象）

### デバッグ方針
- `layoutDebugOverlay` フラグで枠線・基準線を描画可能にする
- STEP3検証はオーバーレイONで実施し、配置確定後にOFFで最終確認する
