# Marathon Coach Data Field (Connect IQ) - AGENT.md

## 目的
市民マラソン向けの **Connect IQ データフィールド（1画面）** を実装する。  
通常の「数値の羅列」ではなく、**数値から導かれる“次のアクション”をコーチカードとして提示**する。

- 主対象：PB 5:00前後のランナー
- レース中は判断負荷を最小化し、事故（オーバーペース／補給遅れ／脱水兆候）を減らす

## 現在スコープ（暫定）
- 当面は **通常版（Core）を完璧に仕上げることを最優先** とする。
- モードは **通常版 / カスタムモード** の2つを対象とする。
- **後半レースモードは実装しない**。

---

## 成果物
- Connect IQ Data Field（1面）
- 主要ロジック：
  - 心拍上限（Garmin心拍ゾーン基準）
  - 補給タイマー（LAPでリセット）
  - 補給間隔35分固定（通常版）
  - カスタムモードのみ補給間隔を個別調整
  - ドリフト検知（基準区間20–30分）
  - ACTION/FUELの交互表示（ただし特定状態では固定）

---

## UI方針（コーチカード風 / B案）
- 画面中央に「コーチカード」を表示し、**短い文言**で行動指示を出す（日本語/英語対応）
- 数値は補助。視線の主役は「カード」
- **右下：いまのぺーす**（10秒移動平均）
  - 数字は上寄せ、単位 `/km` は下に小さく

### 画面内の要素（1画面）
- 中央：コーチカード（状態に応じて文言・色・アイコン）
- 左上付近：心拍（HR）とゾーン（ZONE）
  - Garmin心拍ゾーン取得時：`HR/Zn`（例 `152 / Z3`）を表示
  - ゾーン取得不可時：`HR / --`
- 右上：FUELリング＋残り時間
- 右下：いまのぺーす（10秒平均）＋ `/km`（小）
- 下段：距離（DIST）／経過時間（TIME）
- 目標との差（PACE Δ）：表示（ただし数値羅列感を抑え、短く）

## 通常版 / カスタムモード（現行方針）
本アプリは **通常版（Core）** を基盤にし、必要に応じて **カスタムモード** で個別調整を行う。  
**2モード（通常版/カスタムモード）以外の分岐は持たない**。また、**後半レースモードは実装しない**。

### 通常版（Core）
目的：信頼できる“土台”を提供し、判定のバタつきを抑える。

含まれるもの：
- コーチカード（ちょい上げ／ちょい落とし／そのまま／補給系）
- 距離通知カード（1kmごと + 種目別の節目/ゴール）
- 右下：いまのぺーす（10秒平均、単位は下に小さく）
- 距離／経過時間
- 心拍ZONE上限（Garmin心拍ゾーン基準、距離帯で段階）
- ドリフト検知（基準20:00〜30:00、10分窓）＋ DRIFT ONで「水＋補給」固定
- 補給タイマー（ACTION/FUEL交互表示）＋ LAPで補給リセット
- 補給間隔：**35分固定**（LAP時点から再カウント）

### カスタムモード（個別最適化 + サポート）
目的：ユーザー特性に合わせて通常版パラメータを調整する。

個別最適化の対象（例）：
- **補給間隔の調整**（30–40分レンジ、前倒し幅、距離ゲート等）
- ZONE段階の差分（守り/攻め）
- ドリフト閾値、解除条件、クールダウン
- 通知の強さ、頻度、固定表示時間

配布方法：
- **カスタムコード**（1つのコード入力で一括反映）
- 受領後、設定画面でコード入力して反映

サポート（任意）：
- ヒアリング → 設定/当日プラン納品 → 微調整 → レース後レビュー

## 距離通知（現在実装）
距離通知カードは **3行固定フォーマット**。
- 1行目：距離ラベル（例 `8km` / `HALF` / `FULL`）
- 2行目：意味づけ
- 3行目：短い行動メッセージ

### 種目別ルール
- フルマラソン：
  - 1kmごとに通常スプリット
  - `21.1km` で節目カード `HALF`
  - `42.2km` でゴールカード `FULL`
- ハーフマラソン：
  - 1kmごとに通常スプリット
  - `10km` で節目カード `10km`
  - `21.1km` でゴールカード `HALF`
- 10km：
  - 1kmごとに通常スプリット
  - `5km` で節目カード `5km`
  - `10km` でゴールカード `10km`
- 5km：
  - 1kmごとに通常スプリット
  - 中間節目カードなし
  - `5km` でゴールカード `5km`

### 通常スプリット文言
- 序盤 / 中盤 / 後半の3距離帯テンプレートを使用
- 各距離帯に3セットを持ち、`(km - 1) % 3` でローテーション

### 判定と表示
- 到達判定：`distance >= target - 0.02km`
- 表示時間：**7秒**
- 表示優先順位：
  1. `FUEL NOW`
  2. `HR OVER`
  3. `DRIFT ON`
  4. 距離通知カード
  5. 通常カード（ACTION/FUEL）
- 警告カード表示中に到達した距離通知はスキップする

## データ定義（内部）
- `raceDistanceKm`（設定）
- `targetTimeSec`（設定）
- `targetPaceSecPerKm` = `targetTimeSec / raceDistanceKm`
- `fuelIntervalSec`（通常版は固定 `35 * 60`。カスタムモードのみ個別値）
- `lastFuelTimeSec`（LAP押下で更新）
- `fuelDueTimeSec` = `lastFuelTimeSec + fuelIntervalSec`

### ペース（表示用）
- `paceNowSecPerKm`：**10秒移動平均**
- `paceDeltaSecPerKm` = `paceNowSecPerKm - targetPaceSecPerKm`

## 補給判定の整合ルール（カード / 右上メーター）
- 補給間隔は `35:00` 固定。`Lap` 押下で `35:00` へリセットする。
- 残り `> 2:00`：メーターは **通常**、カードは通常表示（FUELトグル対象外）。
- 残り `2:00〜0:01`：メーターは **注意**、カードは `ACTION/FUEL` を3秒トグルする。
- 残り `<= 0:00`（期限到達/超過、Lapまで継続）：メーターは **警告**、カードは **FUEL NOW固定（最優先）**。
- 右上メーター警告文言は固定で `補給now / 補給後LAP`（英語: `Fuel now / Press LAP`）。

---

## 優先順位（表示固定ルール）
最優先から順に適用し、該当すればコーチカードを固定する。

1. **FUEL NOW（期限超過）**
   - カード：`いま 補給`
2. **HR > ZONE上限（心拍上限超過）**
   - カード：`ちょい落とし`
3. **DRIFT ON（ドリフト検知）**
   - カード：`水＋補給`
4. 通常
   - トグル表示（ACTION ↔ FUEL）

---

## レースプロファイル判定（現在実装）
`raceDistanceKm` の設定値でプロファイルを決める。

- `SHORT`：`raceDistanceKm <= 10.5`（5km/10km想定）
- `HALF`：`abs(raceDistanceKm - 21.0975) <= 0.25`
- `FULL`：上記以外（フルマラソン想定）

## ZONE（心拍上限）ロジック（Garmin心拍ゾーン基準）
距離の絶対値ではなく、`progress = elapsedDistance / raceDistanceKm` の進捗率で5フェーズに分割する。

- `S1`: `< 24%`
- `S2`: `24%〜59%`
- `S3`: `59%〜83%`
- `S4`: `83%〜95%`
- `S5`: `95%〜Finish`

許容ゾーン（`allowedZoneNumber`）とオフセット（`allowedZoneOffsetBpm`）：

| Profile | S1 | S2 | S3 | S4 | S5 |
| --- | --- | --- | --- | --- | --- |
| FULL | Z2 +0 | Z3 +0 | Z3 +2 | Z4 +0 | Z4 +3 |
| HALF | Z3 +0 | Z4 -2 | Z4 +0 | Z4 +2 | Z4 +4 |
| SHORT | Z4 +0 | Z5 +2 | Z5 +3 | Z5 +4 | Z5 +5 |

HR超過判定（`HR > allowed + 1bpm`）：
- Over開始継続秒数：
  - `S1/S2/S3`: `12秒`
  - `S4`: `10秒`
  - `S5`: `20秒`
- Over解除：
  - 回復継続 `5秒`
  - 解除閾値は `S4` のみ `allowed -1bpm`、それ以外は `allowed -2bpm`

---

## ACTION判定（現在実装）
「ちょい上げ」は、遅れがあり、かつ心拍余裕があり、Cardiac Costが悪化していない場合のみ発火する。

- フェーズ定義は ZONE と同じ（`S1..S5`）。
- Cardiac Cost比：
  - `baseCost = baseHR * basePace`
  - `curCost = curHR * curPace`
  - `ccRatio = curCost / baseCost`
  - `ccRatio` は次の条件を満たす場合のみ有効（それ以外は `null` 扱い）：
    - ドリフト基準確定済み（`baseHR/basePace` が有効）
    - rollingサンプル数 `>= 30`
    - `abs(curPace - basePace) <= 5秒/km`（pace stability 条件）

- 前提条件（1つでも満たさない場合は発火しない）：
  - FUEL期限超過でない
  - HR超過状態でない
  - DRIFT ONでない
  - `paceNowSecPerKm` / `targetPaceSecPerKm` / `allowedMaxHeartRate` / `currentHeartRate` が有効
- ちょい上げ発火閾値（`paceDelta >=` かつ `headroom >=`）：

| Profile | S1 | S2 | S3 | S4 | S5 |
| --- | --- | --- | --- | --- | --- |
| FULL | `+12s, 8bpm` | `+8s, 6bpm` | `+6s, 4bpm` | `+4s, 3bpm` | `+3s, 2bpm` |
| HALF | `+10s, 7bpm` | `+6s, 5bpm` | `+4s, 3bpm` | `+3s, 2bpm` | `+2s, 1bpm` |
| SHORT | `+8s, 6bpm` | `+5s, 4bpm` | `+3s, 2bpm` | `+2s, 1bpm` | `+1s, 0bpm` |

- Cardiac Costゲート（ちょい上げ時）：
  - `FULL`: `ccRatio <= 1.06`
  - `HALF`: `ccRatio <= 1.08`
  - `SHORT`: `ccRatio <= 1.10`
  - `ccRatio == null` の場合はゲート未適用（発火可）
- ヒステリシス：
  - ON：条件成立が `6秒` 継続
  - OFF：`paceDelta < (閾値 -2秒)` または `headroom < (閾値 -1bpm)` または Cardiac Costゲート不成立が `5秒` 継続
- ちょい落とし（Ease）判定：
  - `paceDelta <= -8秒/km` または `headroom <= 閾値` または `baselineHrDelta >= 閾値` または `ccRatio >= 閾値`
  - `headroom` / `baselineHrDelta` / `ccRatio` の閾値はプロファイル・フェーズ別に可変
    - `ccRatio` 閾値：`FULL 1.10 / HALF 1.12 / SHORT 1.15`

## ドリフト検知（確定：方式B）
### 基準区間（Baseline）
- レース開始後、**20:00経過 かつ 3.0km到達** を満たした時点から連続10分で基準を作る
  - `basePace`（平均ペース）
  - `baseHR`（平均心拍）
- 基準確定前（ACTION判定に入る前）は、コーチカードにウォームアップ文言を表示する
  - 文言はローテーション表示し、切替間隔は **15秒**
  - 基準確定後は通常のACTION判定文言（ちょい上げ / そのまま / ちょい落とし）へ切り替える

### 判定窓（Rolling）
- 直近10分：
  - `curPace`（平均ペース）
  - `curHR`（平均心拍）

### ペース安定条件
- `abs(curPace - basePace) <= 5秒/km` の場合のみ判定する

### ドリフトON/OFF
- ON：`curHR - baseHR >= +10 bpm`
- OFF：`curHR - baseHR <= +6 bpm` を **60秒継続**

### UI
- ドリフトON時：コーチカードを **「水＋補給」固定**（トグル停止）

## STEP3レイアウト方針（確定）
### 対象デバイス
- 基準：`fr255`
- デバイス差分は機種個別分岐ではなく、`small / medium / large` のサイズクラスで吸収する
  - `small`: `min(width, height) <= 218`
  - `medium`: `219 .. 260`（`fr255`はここ）
  - `large`: `>= 261`

### STEP3で固定する範囲
- 固定対象は **外枠（safe area）+ 内部分割（中央割 + 縦4分割）**
- ロジック（判定、トグル、固定表示の条件）はSTEP4以降で実装する

### レイアウトモデル（中央割 + 縦4分割）
- 分割ルール：画面を中央で左右に分け、さらに上下を4段に分ける（計8セル）
- 添付モック（`assets/step3_layout_fr255_4block_mockup.png`）準拠で配置する
- 要素配置：
1. 1段目（上段）：左=`HR/ZONE`、右=`FUELリング上部（右1-2段跨ぎ）`
2. 2段目：左=`CARD（上半）`、右=`FUELリング下部（右1-2段跨ぎ）`
3. 3段目：左=`CARD（下半）`、右=`PACE（大） + /km`
4. 4段目（下段）：左=`DIST`、右=`TIME`（`PACE Δ` は下段内で補助表示）

### 座標・サイズの決め方
- 画面絶対値ではなく **比率ベース** を使う
- 主要寸法は `ratio + min/max px` のガードで定義する
  - 例：`outerPadding = 5%`（`min 8px / max 16px`）
- ブロック間ギャップも比率で定義し、クラス別に微調整する

### 表示優先順位（狭小時）
- 欠けを避ける優先順は以下とする
1. コーチカード文言（CARD）
2. 現在ペース（PACE）
3. HR/ZONE（1段目・左セル）
4. FUEL残り時間（右1-2段跨ぎセル）
5. DIST/TIME（4段目・左右セル）
6. PACE Δ（最初に縮退/省略対象）

### デバッグ方針
- `layoutDebugOverlay` フラグで枠線・基準線を描画可能にする
- STEP3検証はオーバーレイONで実施し、配置確定後にOFFで最終確認する
- 配置イメージ参照: `assets/step3_layout_fr255_4block_mockup.png`
